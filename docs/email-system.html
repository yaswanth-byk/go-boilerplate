Email System

Transactional emails sent via Resend, rendered from Go HTML templates, and processed asynchronously through the job queue.

Files Involved

FilePurposeWhy It Exists
email/client.goCreates the Resend API client. Provides a SendEmail() method that renders an HTML template with data and sends it via Resend.Centralized email sending. Any part of the app that needs to send an email calls this client.
email/emails.goConvenience methods for each email type (e.g., SendWelcomeEmail). Wraps SendEmail() with the correct template and data.Individual email methods make the API clean: client.SendWelcomeEmail(to, name) instead of manually specifying templates and data each time.
email/templates.goDefines template name constants (e.g., TemplateWelcome = "welcome").Type-safe template references. Using constants instead of strings prevents typos.
email/preview.goProvides sample data for each template, used for email preview/testing.When developing templates, you can render them with realistic test data without sending actual emails.
template/emails/*.htmlThe actual HTML email templates using Go's html/template syntax.Templates are separate files for easy editing by non-developers. They use {{.VariableName}} placeholders.

How Email Sending Works

1

Handler enqueues email job

Your handler (or service) creates an email task and enqueues it to the Asynq job queue. The HTTP response returns immediately — the user doesn't wait for the email to send.

2

Asynq worker picks up the task

The background job server dequeues the task and calls the email handler function.

3

Template rendered

The email client loads the HTML template from template/emails/, executes it with the provided data (Go's html/template), and produces the final HTML.

4

Email sent via Resend

The rendered HTML is sent through the Resend API.

Adding a New Email

Step 1: Create the HTML template

<!-- template/emails/password_reset.html -->
<!DOCTYPE html>
<html>
<body>
  <h1>Reset Your Password</h1>
  <p>Hi {{.UserFirstName}},</p>
  <p>Click the link below to reset your password:</p>
  <a href="{{.ResetLink}}">Reset Password</a>
  <p>This link expires in 1 hour.</p>
</body>
</html>

Step 2: Register the template name

// email/templates.go
const (
    TemplateWelcome       Template = "welcome"
    TemplatePasswordReset Template = "password_reset"  // Add this
)

Step 3: Add preview data

// email/preview.go
var PreviewData = map[string]map[string]string{
    "welcome": {"UserFirstName": "John"},
    "password_reset": {                          // Add this
        "UserFirstName": "John",
        "ResetLink":     "https://example.com/reset?token=abc123",
    },
}

Step 4: Create the convenience method

// email/emails.go
func (c *Client) SendPasswordResetEmail(to, firstName, resetLink string) error {
    return c.SendEmail(
        to,
        "Reset Your Password",
        TemplatePasswordReset,
        map[string]string{
            "UserFirstName": firstName,
            "ResetLink":     resetLink,
        },
    )
}

Step 5: Create the background task

See the Background Jobs page for how to create and register the Asynq task.

Template Variables

Templates use Go's html/template syntax. Variables are passed as a map[string]string:

// In the template:
<p>Hi {{.UserFirstName}},</p>

// Passed as:
data := map[string]string{
    "UserFirstName": "John",
}
💡 Resend setup

Get your Resend API key from resend.com/api-keys and set it as BOILERPLATE_INTEGRATION.RESEND_API_KEY. The default sender address is onboarding@resend.dev — update it in client.go for production.
