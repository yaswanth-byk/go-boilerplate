Getting Started

Everything you need to get the boilerplate running on your local machine in under 10 minutes.

Prerequisites

You need the following tools installed before proceeding. Each one plays a specific role:

ToolMinimum VersionWhy You Need ItInstall
Go≥ 1.21 The backend is written entirely in Go. The Go toolchain compiles, runs, and tests the application. go.dev/dl
Node.js≥ 22 Required for running the TypeScript packages (Zod schemas, OpenAPI generator, email templates). nodejs.org
Bun≥ 1.3 The JavaScript/TypeScript package manager and runtime. Much faster than npm/yarn. Used by the monorepo. curl -fsSL https://bun.sh/install | bash
PostgreSQL≥ 15 The primary database. All application data is stored here. You can use a local install or a managed service. brew install postgresql or pgAdmin
Redis≥ 7 Used as the backend for the Asynq job queue (background task processing) and for caching. brew install redis
Task≥ 3 A task runner (like Make but simpler). Used to run, build, test, and lint the Go backend. brew install go-task
DockerLatest Only needed for running tests. Testcontainers spins up real PostgreSQL instances in Docker. docker.com
golangci-lintLatest A fast Go linters aggregator. Runs dozens of linters in parallel to catch bugs and style issues. brew install golangci-lint

Step 1: Clone the Repository

git clone https://github.com/yaswanth-byk/go-boilerplate.git
cd go-boilerplate

Step 2: Install Dependencies

TypeScript packages (monorepo-wide)

This installs all Node.js dependencies for the TypeScript packages (Zod, OpenAPI generator, email templates) and dev tools (Turborepo, TypeScript).

bun install

Go modules

This downloads all Go dependencies defined in go.mod. They're cached locally in your Go module cache.

cd apps/backend
go mod download

Step 3: Create the Database

The application expects a PostgreSQL database named boilerplate. You can create it in one of several ways:

Option A: Command line

createdb boilerplate

Option B: SQL statement

-- In psql or any SQL client:
CREATE DATABASE boilerplate;

Option C: pgAdmin

Right-click "Databases" → "Create" → "Database" → name it boilerplate (all lowercase).

⚠️ Important

The database name must be exactly boilerplate in lowercase. PostgreSQL is case-sensitive when names are quoted. Creating "Boilerplate" in pgAdmin (with quotes) will result in a "database not found" error.

Step 4: Configure Environment Variables

Copy the sample config and customize it:

cd apps/backend
cp .env.sample .env

Open .env in your editor. Here are the critical values you must set:

VariableWhat to SetRequired?
BOILERPLATE_DATABASE.PASSWORD Your PostgreSQL password for the postgres user Required
BOILERPLATE_AUTH.SECRET_KEY Your Clerk secret key (starts with sk_test_ or sk_live_) Required
BOILERPLATE_INTEGRATION.RESEND_API_KEY Your Resend API key (starts with re_) Required
BOILERPLATE_REDIS.ADDRESS Redis URL, usually redis://localhost:6379 Required
BOILERPLATE_OBSERVABILITY.NEW_RELIC.LICENSE_KEY Your New Relic license key (leave empty to disable) Optional

See the Configuration page for a full reference of every variable.

Step 5: Start the Services

Make sure PostgreSQL and Redis are running

# macOS (Homebrew)
brew services start postgresql
brew services start redis

# Verify they're running
pg_isready      # Should say "accepting connections"
redis-cli ping  # Should say "PONG"

Start the backend

cd apps/backend
task run

You should see output like:

2026-03-06 10:00:00 INF starting server port=8080 env=local

Start TypeScript packages in dev mode (optional)

# From the project root
bun run dev

Step 6: Verify It Works

Health check

curl http://localhost:8080/status

Expected response:

{
  "status": "healthy",
  "timestamp": "2026-03-06T15:00:00Z",
  "environment": "local",
  "checks": {
    "database": { "status": "healthy", "response_time": "1.5ms" },
    "redis":    { "status": "healthy", "response_time": "0.8ms" }
  }
}

API documentation

Open http://localhost:8080/docs in your browser to see the interactive OpenAPI documentation.

Common Issues

ErrorCauseFix
database "boilerplate" does not exist The PostgreSQL database hasn't been created yet Run createdb boilerplate
password authentication failed Wrong password in .env Update BOILERPLATE_DATABASE.PASSWORD
connection refused (port 5432) PostgreSQL is not running brew services start postgresql
connection refused (port 6379) Redis is not running brew services start redis
task: No Taskfile found You're not in the apps/backend directory cd apps/backend then retry
