Middleware

The 9-layer middleware pipeline processes every request before it reaches your handler. Each layer adds a specific capability.

Files Involved

FilePurposeWhy It Exists
middleware.goCreates the Middlewares struct and its constructor. This is the entry point that aggregates all middleware modules.Groups all middleware into a single struct that can be passed to the router.
rate-limit.goIn-memory rate limiter (20 req/sec per client) with New Relic custom events for rate limit hits.Prevents abuse. Without it, a single client could overwhelm the server.
request_id.goGenerates or extracts a UUID for each request. Stores it in context and returns it in response headers.Request IDs let you trace a single request through all logs, even across services.
tracing.goAdds custom attributes (IP, user agent, request ID, route) to the New Relic transaction.Enriches New Relic data so you can filter and search by IP, user, or route in the dashboard.

global.goGlobal Echo error handler. Catches all errors (including panics) and converts them to the standard JSON format.Ensures every error — whether from your code, Echo, or a panic — returns a consistent JSON response.

Pipeline Order

Middleware runs in the exact order shown below. The order matters — for example, the Request ID must run before the Context Enhancer so the logger can include it.

1

Rate Limiter

rate-limit.go

Uses Echo's built-in rate limiter middleware configured with an in-memory store. Limits each client IP to 20 requests per second. When exceeded, immediately returns:

HTTP 429 Too Many Requests
{
  "code": "RATE_LIMIT_EXCEEDED",
  "message": "Too many requests",
  "status": 429
}

Also sends a RateLimitExceeded custom event to New Relic with the client IP and request path.

2

CORS

Configured via BOILERPLATE_SERVER.CORS_ALLOWED_ORIGINS. Sets Access-Control-Allow-Origin, Allow-Methods, Allow-Headers, and Allow-Credentials. Handles preflight OPTIONS requests automatically.

3

Security Headers

Uses Echo's Secure middleware. Adds standard security headers: X-Frame-Options: SAMEORIGIN, X-Content-Type-Options: nosniff, X-XSS-Protection: 1; mode=block.

4

Request ID

request_id.go

Checks for an X-Request-ID header. If present and a valid UUID, uses it. Otherwise generates a new UUID v4. The ID is:

  • Stored in the Echo context (key: request_id)
  • Added to the response X-Request-ID header
  • Used by all downstream middleware and handlers
5

New Relic Transaction

The nrecho-v4 integration middleware starts a New Relic transaction for this request. Automatically captured: method, URL, status code, response time, errors.

6

Enhanced Tracing

tracing.go

Adds custom attributes to the New Relic transaction:

  • request.id — the unique request ID
  • request.ip — client IP address
  • request.userAgent — browser/client user agent
  • request.userId — authenticated user ID (if available)
7

Context Enrichment

context.go

Creates a new Zerolog logger for this specific request. The logger is pre-populated with all available context fields. Every log statement within this request automatically includes this context — you never need to manually add request ID, user ID, etc.

Context fields added: request_id, method, path, ip, user_id, trace.id, span.id, entity.name, entity.type, hostname.

8

Request Logger

Logs the completed request (runs after the handler returns). Includes: status code, latency, method, URI, client IP, and any error. Uses appropriate log level:

  • Info for 2xx responses (success)
  • Warn for 4xx responses (client errors)
  • Error for 5xx responses (server errors)
9

Panic Recovery

Wraps the handler in a recover(). If your handler code panics (e.g., nil pointer dereference), this catches it and returns a 500 error instead of crashing the entire server process.

The Middlewares Struct

type Middlewares struct {
    Auth    *auth         // Clerk JWT validation
    server  *server.Server
}

func NewMiddlewares(s *server.Server) *Middlewares {
    return &Middlewares{
        Auth:   newAuth(s),
        server: s,
    }
}

The struct is intentionally lean. Each middleware is a method on Middlewares (for pipeline middleware) or on the auth sub-struct (for route-level middleware).

Global Error Handler

global.go

The global error handler is registered via echo.HTTPErrorHandler. It catches every error and:

  1. Checks if it's an HTTPError (our custom type) → returns that directly
  2. Checks if it's an echo.HTTPError (from Echo's built-in errors) → maps to our format
  3. Falls back to a generic 500 Internal Server Error

It also logs every error with full context (request ID, user ID, stack trace, error code) and reports it to New Relic.

Adding Your Own Middleware

// 1. Create a new file: internal/middleware/my_middleware.go
func (m *Middlewares) MyMiddleware() echo.MiddlewareFunc {
    return func(next echo.HandlerFunc) echo.HandlerFunc {
        return func(c echo.Context) error {
            // Before handler
            c.Set("my_data", "hello")

            err := next(c) // Call the next handler

            // After handler
            return err
        }
    }
}

// 2. Add it to the pipeline in router.go:
router.Use(middlewares.MyMiddleware())
