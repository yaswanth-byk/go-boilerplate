Configuration

How the application loads, validates, and uses its configuration. Every environment variable, what it controls, and how the Config struct is assembled.

Files Involved

FilePurpose
config.goDefines all config structs (Config, ServerConfig, DatabaseConfig, etc.), loads env vars via koanf, and validates using go-playground/validator.
observability.goDefines ObservabilityConfig with sub-configs for logging, New Relic, and health checks. Provides sensible defaults and validation.
.env.sampleTemplate file showing every available environment variable with example values. Copy this to .env for local development.

How Config Loading Works

Configuration loading happens in config.LoadConfig() and follows four sequential steps:

1

Auto-load .env file

The import _ "github.com/joho/godotenv/autoload" automatically reads the .env file when the program starts. This is a blank import — including it is enough to trigger the loading. This means your .env file is read before any Go code runs.

2

Read environment variables with koanf

koanf reads all environment variables that start with BOILERPLATE_, strips the prefix, and converts dots to nested keys. For example:
BOILERPLATE_DATABASE.HOST → database.host

3

Unmarshal into typed struct

The flat key-value map is unmarshalled into the Config struct using koanf's struct unmarshaller. Each field's koanf tag maps to the env var name.

4

Validate with go-playground/validator

Every field with a validate:"required" tag is checked. If any required field is missing, the application panics with a clear error message showing which fields failed validation.

The Config Struct

type Config struct {
    Primary       Primary              `koanf:"primary" validate:"required"`
    Server        ServerConfig         `koanf:"server" validate:"required"`
    Database      DatabaseConfig       `koanf:"database" validate:"required"`
    Auth          AuthConfig           `koanf:"auth" validate:"required"`
    Redis         RedisConfig          `koanf:"redis" validate:"required"`
    Integration   IntegrationConfig    `koanf:"integration" validate:"required"`
    Observability *ObservabilityConfig `koanf:"observability"`
}
💡 Note

Observability is a pointer (*ObservabilityConfig) and not required. If no observability env vars are set, it defaults to sensible values via DefaultObservabilityConfig(). All other sections are required — the app won't start without them.
