Authentication

JWT-based authentication via Clerk. The middleware validates tokens, extracts session claims, and stores user context for downstream handlers.

Files Involved

FilePurposeWhy It Exists
middleware/auth.goDefines the RequireAuth middleware that validates Clerk JWTs and extracts session claims.Protects endpoints that require a logged-in user. Extracts user ID, role, and permissions from the JWT.
service/auth.goInitializes the Clerk SDK with the secret key from config.Clerk needs the secret key set once at startup. This service handles that initialization.

What is Clerk?

Clerk is a third-party authentication service that handles user registration, login, session management, and organization features. Your frontend uses Clerk's SDKs to authenticate users — Clerk issues a JWT — and this backend validates that JWT on protected routes.

How Auth Works

1

Frontend sends JWT

The frontend includes the Clerk session token in the Authorization: Bearer <token> header.

2

Middleware extracts the token

The RequireAuth middleware reads the Authorization header, strips the Bearer prefix, and gets the raw JWT string.

3

Clerk SDK verifies the JWT

The Clerk SDK validates the JWT signature against Clerk's public keys, checks expiration, and returns the session claims.

4

Claims are stored in context

The middleware extracts user ID, organization role, and permissions from the JWT claims, and stores them in the Echo context.

5

Handler accesses user info

Your handler can read c.Get("user_id"), c.Get("user_role"), etc. from the context.

Context Values Set by Auth Middleware

KeyTypeDescription
user_idstringThe Clerk user ID (e.g., user_2abc123...)
user_rolestringThe user's role in the active organization (e.g., admin, member)
permissionsinterface{}Permissions associated with the user's organization membership

Protecting Routes

// In router.go:

// Public routes — no auth needed
router.GET("/status", healthHandler.CheckHealth)

// Protected routes — auth required
authGroup := router.Group("/api/v1", middlewares.Auth.RequireAuth)
authGroup.GET("/profile", handler.Handle(h.User.Handler, h.User.GetProfile, 200, &GetProfileReq{}))
authGroup.POST("/orders", handler.Handle(h.Order.Handler, h.Order.Create, 201, &CreateOrderReq{}))

Accessing User Info in Handlers

func (h *UserHandler) GetProfile(c echo.Context, req *GetProfileReq) (*ProfileRes, error) {
    userID := c.Get("user_id").(string)
    role := c.Get("user_role").(string)

    // Use userID to fetch user data from the database
    user, err := h.services.User.GetByClerkID(userID)
    if err != nil {
        return nil, err
    }
    return &ProfileRes{Name: user.Name, Role: role}, nil
}

💡 Getting your Clerk secret key

Go to dashboard.clerk.com → your application → "API Keys" → copy the "Secret key". It starts with sk_test_ (development) or sk_live_ (production). Set it as BOILERPLATE_AUTH.SECRET_KEY in your .env file.
