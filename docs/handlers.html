Handlers

The handler layer uses Go generics to eliminate boilerplate. Write your business logic; the framework handles binding, validation, logging, and tracing.

Files Involved

The Handler Base Struct

Every handler group embeds the Handler base struct:

type Handler struct {
    server   *server.Server
    services *service.Services
}

It gives every handler access to the server (DB, Redis, config, logger) and services (business logic).

Three Generic Handler Functions

Instead of writing raw Echo handlers, you use one of three generic functions that handle all the boilerplate:

1. Handle[Req, Res] — Returns JSON

For endpoints that return a JSON response body.

func Handle[Req Validatable, Res any](
    base Handler,                                        // Base handler with dependencies
    fn func(echo.Context, *Req) (*Res, error),           // Your business logic
    statusCode int,                                      // HTTP status code on success
    req *Req,                                            // Empty request struct for binding
) echo.HandlerFunc

What it does automatically:

  1. Logs "handling request" with the handler name and route
  2. Binds the request body/params/query to your Req struct
  3. Calls Validate() on the struct → returns 400 with field errors if invalid
  4. Logs "validation successful" with duration
  5. Calls your function fn(c, req)
  6. If error → handles it (returns appropriate HTTP error)
  7. If success → returns statusCode with the Res as JSON
  8. Logs "request completed" with total duration
  9. Adds New Relic attributes for handler name, validation time, handler time

2. HandleNoContent[Req] — Returns no body

For endpoints that return an empty body (e.g., 204 No Content, 202 Accepted).

func HandleNoContent[Req Validatable](
    base Handler,
    fn func(echo.Context, *Req) error,    // No response return type
    statusCode int,
    req *Req,
) echo.HandlerFunc

3. HandleFile[Req] — Returns a file download

For endpoints that return a file (PDF, CSV, etc.).

func HandleFile[Req Validatable](
    base Handler,
    fn func(echo.Context, *Req) ([]byte, error),  // Returns file bytes
    statusCode int,
    req *Req,
    filename string,     // Download filename
    contentType string,  // MIME type
) echo.HandlerFunc

Building a New Handler

Follow this step-by-step guide to add a new handler.

Step 1: Define the request struct

The request struct must implement the Validatable interface (it needs a Validate() method).

// internal/handler/user.go

type CreateUserRequest struct {
    Name  string `json:"name" validate:"required,min=2,max=100"`
    Email string `json:"email" validate:"required,email"`
}

func (r *CreateUserRequest) Validate() error {
    return validator.New().Struct(r)
}

Step 2: Define the response struct

type UserResponse struct {
    ID        string    `json:"id"`
    Name      string    `json:"name"`
    Email     string    `json:"email"`
    CreatedAt time.Time `json:"created_at"`
}

Step 3: Write the handler function

type UserHandler struct {
    Handler  // Embed base handler
}

func (h *UserHandler) Create(c echo.Context, req *CreateUserRequest) (*UserResponse, error) {
    user, err := h.services.User.Create(req.Name, req.Email)
    if err != nil {
        return nil, err  // sqlerr.HandleError auto-maps DB errors
    }
    return &UserResponse{
        ID:        user.ID.String(),
        Name:      user.Name,
        Email:     user.Email,
        CreatedAt: user.CreatedAt,
    }, nil
}

Step 4: Register in handlers.go

type Handlers struct {
    Health  *HealthHandler
    OpenAPI *OpenAPIHandler
    User    *UserHandler       // Add this
}

func NewHandlers(s *server.Server, services *service.Services) *Handlers {
    base := newHandler(s, services)
    return &Handlers{
        Health:  &HealthHandler{Handler: base},
        OpenAPI: &OpenAPIHandler{Handler: base},
        User:    &UserHandler{Handler: base},  // Add this
    }
}

Step 5: Register the route

// router.go
v1 := router.Group("/api/v1", middlewares.Auth.RequireAuth)
v1.POST("/users", handler.Handle(
    h.User.Handler,
    h.User.Create,
    http.StatusCreated,
    &handler.CreateUserRequest{},
))
💡 Tip

You don't need to write any request binding, validation, logging, or error handling code. The Handle function does all of that for you. Just focus on your business logic.

Existing Handler: Health Check

health.go

The health check endpoint (GET /status) verifies database and Redis connectivity:

  • Pings PostgreSQL and measures response time
  • Pings Redis and measures response time
  • Returns overall status: "healthy" if both pass, "unhealthy" if either fails
  • Logs and reports to New Relic on failures
  • Returns the current environment name and timestamp
